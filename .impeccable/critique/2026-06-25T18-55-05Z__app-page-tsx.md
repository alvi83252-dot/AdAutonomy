## Anti-Patterns Verdict

**LLM assessment:** Yes — in its current skin this reads as AI-generated. The cinematic *ambition* is right for a hackathon hero, but the *vocabulary* is the exact 2025–26 AI signature: indigo→violet gradients, a `bg-clip-text` gradient wordmark, glow buttons, and a purple/blue radial-gradient parallax background. The brief asked for "premium, distinctive, alive" — the execution reaches for the most common generative defaults, which undercuts distinctiveness.

**Deterministic scan (bundled detector, app/ + components/):** 25 findings, all `warning`:
- `ai-color-palette` ×20 — indigo/purple/violet across page.tsx, audience, brief, investor, videos, CreativePreview, LoadingOverlay, Navbar, NavigationProgress, PersonalAssistant, SplashScreen, Stepper, Timeline, button.tsx
- `gradient-text` ×3 — page.tsx hero (L84), Navbar wordmark (L36), SplashScreen (L56)
- `bounce-easing` ×1 — PersonalAssistant `animate-bounce` (L151)
- `side-tab` ×1 — side-stripe/tab accent

**Root cause the detector can't see:** the theme tokens (`--primary: 239 84% 67%`, a perfectly good indigo) exist in `globals.css`, but components ignore them and hardcode `text-indigo-400`, `from-indigo-500 to-purple-600`, `bg-white/10`. So the palette problem and the broken-light-mode problem are the same problem: **the design system is defined but bypassed.**

**Visual overlays:** Not run. No dev server was running and the local Node tooling threw a libuv assertion on this Windows setup, so live browser injection/overlay was skipped. The CLI detector is the deterministic evidence for this run; a live pass can be added after `npm run dev`.

## Overall Impression

The bones are good: a real token system, a coherent component set (AnimatedCard, Stepper, Timeline, well-structured Button variants), genuine motion craft, and a clear information architecture that mirrors the agent pipeline. What holds it back is that it wears the generic AI uniform and only truly exists in dark mode. The single biggest opportunity: **commit to a distinctive palette routed through the tokens** — that one move kills ~23 of 25 detector findings, fixes light mode, and makes the "premium/autonomous company" story land.

## What's Working

1. **Information architecture mirrors the product.** Nav and routes map 1:1 to the agent pipeline (Brief → Creative → Simulation → Deploy). A judge instantly understands the autonomous-company concept.
2. **Status surface is rich.** Stepper, LoadingOverlay, NavigationProgress, and the agent activity Timeline give the app a live, agentic feel — and the Stepper correctly pairs icons with color so state isn't color-only.
3. **Component discipline.** `Button` has real variant coverage and token-based focus rings (`focus-visible:ring-ring`); `AnimatedCard` gives a consistent surface. The system is there to build on.

## Priority Issues

- **[P1] The token system is bypassed by hardcoded indigo/purple.** Components use `text-indigo-400` / `from-indigo-500 to-purple-600` instead of `text-primary` / semantic tokens. **Why it matters:** it's the dominant AI tell (20/25 findings), and it means theme changes don't propagate. **Fix:** define a distinctive brand ramp in OKLCH on the existing `--primary`/accent tokens, then replace hardcoded `indigo-*`/`purple-*` with token classes. *Suggested command: `/impeccable colorize`.*
- **[P1] Light mode is partially broken.** Pervasive `bg-white/10`, `border-white/20`, `text-white`, and `indigo-300`-on-`indigo-500/20` assume a dark background. In light mode the Stepper track (`bg-white/10`) is invisible and active-state contrast drops below 4.5:1. **Why it matters:** the theme toggle ships a visibly broken second mode. **Fix:** swap white-alpha for `border`/`muted`/`foreground` token alphas; verify both themes. *Suggested command: `/impeccable audit`.*
- **[P1] The headline pipeline isn't actually live.** `/api/campaign/run` executes all nine agents server-side in one blocking POST, then returns; the Stepper animates only from the final `currentStep`. **Why it matters:** the core promise ("watch the autonomous loop") is a post-hoc replay, and a slow run shows one overlay with no per-agent progress. **Fix:** stream step events (SSE/streamed response) or run steps client-side via the existing `runSingleStep` so the Stepper advances in real time. *Suggested command: `/impeccable animate` (motion tied to real state).*
- **[P2] No `prefers-reduced-motion` path.** Parallax, spring, zoom, splash, and staggered reveals run unconditionally. **Why it matters:** vestibular-accessibility failure and a skill-mandated requirement. **Fix:** add a `@media (prefers-reduced-motion: reduce)` fallback (crossfade/instant) and gate Framer variants. *Suggested command: `/impeccable animate`.*
- **[P2] Gradient-text wordmark + hero, and generic error copy.** `bg-clip-text` gradients on the brand mark/hero are a banned decorative tell; "Pipeline failed" / "Please fill in all fields" name no cause or next step. **Fix:** solid-color wordmark with weight for emphasis; error messages that say what failed and what to do. *Suggested commands: `/impeccable typeset`, `/impeccable clarify`.*

## Persona Red Flags

**Alex (Power User):** No keyboard shortcuts anywhere. The splash screen and entrance choreography can't be skipped. A launched campaign can't be cancelled (fire-and-forget POST). One rigid linear path with no bulk/replay affordance.

**Sam (Accessibility-Dependent):** No reduced-motion alternative for heavy parallax/spring/zoom motion. Nav `<Link>`s have no visible focus ring (only `Button` does), so keyboard focus is partly invisible. Light-mode active states (`indigo-300` on `indigo-500/20`) and muted-on-tint body text risk failing 4.5:1.

**Jordan (First-Timer / judge):** Lands on motion choreography before content. Data pages surface ROAS/CPC/CTR with no inline explanation. On failure, "Pipeline failed" gives no recovery path. No tooltip or empty state teaching what each agent does.

## Minor Observations

- `animate-bounce` on the assistant typing indicator — swap for an ease-out pulse.
- One `side-tab`/side-stripe accent flagged — replace with a full border or background tint.
- Mobile menu animates `height: auto` (layout property); fine, but `clip-path`/opacity would be smoother.
- `glow` button variant hardcodes the gradient — route it through tokens so it restyles with the brand.

## Questions to Consider

- What would a confident, *non-purple* AdAutonomy look like — terminal-green finance, oxblood/ink editorial, or a single drenched brand color carrying the hero?
- Should the pipeline genuinely stream agent-by-agent, so the "hands-off company" is something you *watch happen* rather than a replay?
- Does a product/tool this task-focused need a splash screen at all, or does that motion belong only on the marketing hero?
